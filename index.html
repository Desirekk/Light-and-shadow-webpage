Light & Shadow (Working Title)

A mobile idle RPG in active development by Grindare Studio, built in Godot 4.5 with a Microsoft PlayFab backend. The project focuses on long-term progression, retention-driven systems, and competitive play.

Platform: Android & iOS Stage: Work in Progress / Pre-Alpha Format: Casual / Idle

Gameplay Video

This video demonstrates the current build’s core systems: UI flow, asynchronous PvE activities, combat resolution, equipment progression, and early competitive elements.

  • Core loop: choose activity → resolve combat → rewards → upgrades → repeat.
  • Designed for short sessions with long-term progression.
  • Backend-supported progression and data storage via PlayFab.

Current Build

  • Playable mobile build with account flow and core gameplay loop implemented.
  • Asynchronous PvE hunts/missions with progression rewards.
  • Equipment progression and stat growth supporting long-term retention.
  • Early PvP/competitive structure and ranking-driven motivation.
  • Prepared foundation for live-ops style iteration (content expansion, balancing, testing).

Major Systems in Development

Beyond what is currently visible in the gameplay video, several high-impact systems remain to be implemented and polished to reach a beta/soft-launch quality milestone.

  • Guilds as a core social and progression pillar: roles, progression, upgrades, guild economy.
  • Guild Wars with large-scale PvP (member-vs-member format inspired by classic idle RPG competitive structures).
  • Dungeons / lochs as repeatable, multi-stage PvE content to deepen progression and retention.
  • Expanded content layers, balancing, QA hardening, and structured testing with external testers.
  • Cohesive visuals, animations and sound design to unify the product experience.

6-Month Funding Plan

The 6-month grant period is designed to convert the current functional build into a stable, content-complete beta candidate, with core social and competitive systems implemented and tested.

  • Months 1–2: Stabilisation, bug fixing, system verification, UI/UX polish, cohesive visuals & animation pass, initial sound integration.
  • Months 3–4: Guild system implementation and progression (upgrades, economy hooks, social layer), first full integration testing.
  • Months 5–6: Dungeons/lochs implementation, guild wars initial version, balancing, external playtesting, performance optimisation, beta readiness.

How the Funding Will Be Used

  • Full-time development to complete missing high-impact systems (guilds, guild wars, dungeons).
  • Visual cohesion: consistent UI assets, animation polish, and production-ready presentation quality.
  • Audio integration: SFX feedback, scene ambience, and progression cues.
  • Structured testing: sourcing testers, running test cycles, fixing defects, and improving stability.
